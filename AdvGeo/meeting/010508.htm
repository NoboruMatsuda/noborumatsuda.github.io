Construction procedure must be revised

  • Implement more general set of construction operators as follows:
  1. Drop perpendicular to a given ray from a given point.
  2. Copy a given distance onto a given ray
  3. Copy a given triangle
  4. Extend given lines to meet
  5. Draw a line parallel to a given line through a given point
  6. Plot a mid point to a given segment.

Those generalized operators might make the readers easier to convince "sufficiency" of the construction operators used in GRAMY.

  • Implement two different procedures for construction and see if they work:

(A) a brute force forward construction

  1. Exhaustive FC to quiescence
  2. non-deterministically apply construction operators shown above (observe branching factors)
  3. Exhaustive FC (observe # successive constructions)

(B) a "constrained" construction

  1. Exhaustive FC to quiescence
  2. non-deterministically apply construction operators shown above
  3. rule out constructions that don't meet the constraints: (observe branching factors)
    • DS configuration totally matches with new figure
    • DS conditions are either unify fact or are quantitatively true in new figure
  4. Exhaustive FC  (observe # successive constructions)

Other concerns

  • We can't say GRAMY could find all possible constructions, because there might be infinitely many redundant constructions.  However, for educational purpose, we can claim that we only allow students to draw constructions with a fixed complexity.
  • Multiple constructions shouldn't run out of memory.  -- will try it again
  • # correct constructions must be reported
  • # construction problems currently appeared in the paper is too small.   Can make construction problems?  Can find more problems?  How about contact with Prof. Seimiya (this is the expert to make construction problems I mentioned in the meeting).
  • Current GRAMY's construction procedure is cognitively black-boxed.  Do we need cognitive model or can we go ahead with the black-box model might be another issue we should concern to design ITS.
