Reorganize a cording schema:

Make them context free so that they can be used for coding protocol data as well as for generating hint messages. 

They must represent (1) a hint type and (2) a content.

Read "Bhaskar, R., & Simon, H. A. (1977). Problem Solving in Semantically Rich Domains: An Example from Engineering Thermodynamics. Cognitive Science, 1(2), 193-215."

It would be interesting to publish a paper on this dual purpose coding schema on hinting.

 

A single line thesis for my dissertation would be:

we have found a theory of hint that fits both protocol data analysis and scaffolding in ITS.

 

The protocol data that we are analyzing now is much more "elicitative," namely, ask more tell less.

Usually, tutoring protocol is more didactic. 

It would be interesting to compare an elicitative ITS and didactic ITS.  (we won't do this for my dissertation, though).

 

H-GOMS on Constraint-Based Student Modeling would be interesting.

Keep going on data analysis with a revised coding schema.
