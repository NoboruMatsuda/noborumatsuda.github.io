Advanced Geometry Tutor pilot test with Kurt

Initiative in proceeding a session

Inference window shows all the inference steps a the beginning

To proceed next step, a student only needs to click [>>] button in the Message window

[OK] button might be thrown away

A pointer must indicate a current inference step in the Inference window

To make it clear which inference steps the student has a responsibility, color each inference step with one of two colors (e.g., red for the tutor's steps and yellow for the student's steps)

Always specify what to do next (instead of just graying things out)

Evan say "Please click [OK]" when the tutor expects to the student to do so

 

Improve scaffolding on an inference step

Show an English description of steps with blanks for step dependent values (i.e., the postulate name, premises, and consequence, etc)

Fill up the blanks when the step is completed

Show a reason of tutor's action for each step

E.g., Say "I'll chose :SSS, because blah blah ..."

Discriminate durable (?) steps, on which something is supposed to be done, and subgoal steps, which is an abstract step to bundle durable steps

subgoal steps can be skipped automatically when problem solving proceeds

Bottom-out hints should go with a beep

 

Make the messages understandable

Clearly distinguish student's tern and tutor's turn

"I'll show you how to do blah ..." for the tutor's turns

"Do blah ..." for the student's turns

Word technical nouns thoroughly

Providing a training "program" for students to get familiar with the technical nouns would be good

Make sure that the student will be aware that a new message appeared in the Message window

Flash?  Type out with an animation? 

 

Fix a bug

Disable to click an item multiple times that is supposed to be clicked only once

 

Other notes

Provide student with papers that has problem figure on it so that they can manipulate overlapping.

Give students pen (not pencil) so that they won't erase their thought process
