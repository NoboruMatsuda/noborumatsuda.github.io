Meeting on AdvGeo (for ITS2000)

Introduction should state ...

  • What is the pedagogical problem:  Reify a complex problem solving strategy.
  • How could such a complex domain be taught?
    • What is backward chain / forward chain (BC/FC)?
    • Basic steps of BC
      • Choose goal G \in Goals & remove G from Goals
      • non deterministically ...
        • match G against the facts in the current state.
        • choose OP whose effect contains G & put preconditions onto Goals
        • if no OP is available, then back-up
    • Three kinds of trees
      • Search tree (OR tree)
      • Proof tree (AND tree)
      • Partial Problem tree (AND/OR tree)
    • What makes the domain difficult to learn
      • nondeterministic procedure -> need search.
      • need to learn cognitive skills to search.
  • Task domain is an advanced geometry theorem proving
  • Best strategy in this domain is BC with some FC at states with new construction.
    • Why BC?  ::  To construct auxiliary lines.
    • Why FC?  ::  To select DS with fewest subgoaling.
    • GRAMY:  better prover than competitors.
      • Has deterministic procedure.
      • Can show students the algorithm for proof.
    • To see how FC/BC works, need to check Schiemes et al results.
  • Previous work on Reification of theorem proving search.
    • Anderson & Koedinger display partial problem space
    • Mixing two kinds of trees  (AND-tree and OR-tree) ->  confusing
    • Too many goals displayed -> dilutes attention
    • Back-up is not clearly reified.
    • ... and so on....
    • Better to check Marcha Lovett@CMU 's work.
  • Our basic approach.
    • "We will address how to design a learning environment for problem solving in a complex domain by reifying the search."
      • Show minimal information required by algorithm.
      • Use common GUI design elements.
      • .... and so on...
    • "All what we need is a search tree so that students can comprehend back-up clearly"
    • Using the depth-limited search is good for both computer and human being.
      • Saves a computer memory to store states (need only b * d)
      • Help human being to access to the states linearly stored.
    • The approach is general so that it should be applied to other domains which need BC as well.
      • e.g.,  Proof tutor @ CMU.
